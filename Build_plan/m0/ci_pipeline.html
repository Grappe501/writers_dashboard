M0.4 CI Pipeline — Gates, Stages, Artifacts

Goal

CI must replicate real builds and catch drift early.

Pipeline Stages

  1. Lint (style + forbidden patterns)
  2. Typecheck (if applicable)
  3. Unit tests (module-level)
  4. Integration tests (module boundaries)
  5. Build (fixtures → outputs)
  6. Compare (outputs vs golden snapshots)
  7. Package (zip/html reader + manifest)
  8. Upload artifacts (reports, manifests, diffs)

Build Order (micro-steps)

  1. Create CI workflow file (GitHub Actions).
  2. Cache dependencies deterministically.
  3. Run fixture builds and compare against golden outputs.
  4. Upload /reports and diffs as CI artifacts.
  5. Block merge on any mismatch unless override log explicitly allows it.

Definition of Done (DoD)

  • CI runs on PR and main.
  • Fixture build outputs are compared against golden snapshots.
  • CI artifacts include reports + manifest + diff summary.
