M4.2 Sequence & Ordering — Deterministic Intra-day Sort

Goal

Within a day, item ordering must be stable across rebuilds.

Sort Keys (example)

  1. explicit timecode (if present)
  2. sequence number (if present)
  3. item_id (tie-breaker)

Build Order (micro-steps)

  1. Implement ordering function and unit tests.
  2. Add exporter that emits ordered lists.
  3. Lock ordering in golden outputs.

Definition of Done (DoD)

  • Rebuild produces identical intra-day ordering on fixtures.
