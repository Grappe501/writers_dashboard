M3.1 Disclosure Modes — Levels, Policies, Unlock Rules

Goal

Define levels (e.g., Strict, Standard, Full) and what each allows.

Baseline Levels

  • Strict: only items up to progress day; no foreshadow; minimal crosslinks.
  • Standard: crosslinks allowed if they do not reveal future outcomes.
  • Full: everything visible; spoiler warnings still available.

Build Order (micro-steps)

  1. Define disclosure policy table.
  2. Implement gating function (item, reader_state) → allow/deny + reason.
  3. Expose reason codes to UI (M7 explain-hidden).

Definition of Done (DoD)

  • Every hidden item provides a reason code and human explanation.
