M1.3 Validation & Conflicts UI — Queue, Explain, Resolve

Goal

Make contradictions actionable without requiring engineers to interpret logs.

Conflict Queue Features

  • Filter by severity (critical/major/minor)
  • Group by rule (timeline collision, entity mismatch, spoiler leak)
  • Show evidence pointers (items + excerpts)
  • One-click actions: edit item, propose merge, request override

Build Order (micro-steps)

  1. Render conflict list from conflict_report.json (or live state).
  2. Add conflict detail view with rule explanation and evidence references.
  3. Provide resolution actions that route to the correct editor screen.
  4. Write resolution status updates (resolved/ignored/overridden) with audit trail.

Definition of Done (DoD)

  • Every conflict has an explanation and evidence links.
  • Resolutions are recorded and survive rebuilds.
