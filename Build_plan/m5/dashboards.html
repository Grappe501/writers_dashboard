M5.2 Dashboards — Conflicts, Overrides, Coverage, Timeline

Goal

One place to see everything blocking a build.

Dashboard Cards

  • Conflicts by severity and rule id
  • Overrides by category and status
  • Coverage: which days have sparse items
  • Entity graph health (dedupe rate, orphan entities)
  • Spoiler gate health (hidden counts by mode)

Build Order (micro-steps)

  1. Build dashboard from reports/manifests.
  2. Add drill-down links into editors and detail pages.
  3. Add export function for reports (zip).

Definition of Done (DoD)

  • All blocking issues are visible and link to resolution actions.
