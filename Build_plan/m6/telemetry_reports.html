M6.10 Telemetry & Reports — Build Stats You’ll Actually Use

Micro-steps

  1. Define `graph_metrics.json`: entity_count, edge_count, alias_count, orphans, cycles, hubs.
  2. Define `dedupe_metrics.json`: candidate_clusters, auto_merges, manual_overrides, review_queue_size.
  3. Emit `build_report.json` with duration timings per stage.
  4. Add thresholds that fail build if metrics exceed safe bounds (optional).

Definition of Done

  1. Reports exist on every build and are deterministic.
  2. Metrics enable quick detection of drift/regressions.
