M6 Overview — Entity Graph & Crosslinking Engine

Goal

  • Provide a deterministic entity graph that powers crosslinks, canon validation, and spoiler-safe retrieval inputs.

Inputs

  • Canon items (M4), schema contracts (M2), UI behaviors (M5).

Outputs

  • entities.json, edges.json, aliases.json, graph_index.json, validation_report.json, crosslinks.json

Micro-steps (build order)

  1. Define entity + edge canonical schemas (versioned).
  2. Implement ID strategy (stable IDs) and slug rules.
  3. Implement alias/dedupe pipeline and audit artifacts.
  4. Build graph deterministically (ordering rules).
  5. Run validation rules and emit validation_report.
  6. Emit crosslinks + backlinks for rendering.
  7. Expose query interfaces consumed by M7.
  8. Add fixtures + tests (ToyBook + Book1Window).
  9. Add telemetry (graph stats, dedupe stats, invalid edge counts).

Definition of Done

  1. Running `build_graph` on the same input produces byte-identical outputs.
  2. All schemas are versioned and validated at load.
  3. Validation_report contains zero ERRORs on fixtures.
  4. Crosslinks/backlinks render deterministically and match golden snapshots.
